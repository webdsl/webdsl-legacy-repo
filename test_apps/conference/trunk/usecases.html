Use Cases in Conference Application

  1. Conference context (c : Conference)
    1. createConference
      1. precondition: principal.isAdmin
      2. postcondition: -

        This should include something saying that a new Conference object was created

    2. invitePcMember
      1. precondition: c.status = assemblePC && securityContext.principal in c.chairs
      2. postcondition: -

        There should be added a new PcInvite to c.pcInvites

    3. acceptPapers
      1. precondition: c.status = assemblePC
      2. postcondition: c.status = acceptingPapers
    4. bidOnPapers

      This is where a PC member bids on a paper

      1. precondition: c.status = bidOnPapers
      2. postcondition: -
    5. Automatically executed use-cases
      1. submissionDeadlineExpired
        1. precondition: c.status = acceptingPapers
        2. postcondition: c.status = bidOnPapers
      2. finishBidding
        1. precondition: And[pcMember.bids.status = bidsCompleted | pcMember : User in c.pc]
        2. postcondition: c.status = decideReviewAssignment
      3. decideReviewAssignment
  2. User context u : User
    1. registerNewUser
      1. precondition: -
      2. postcondition: u.status = registeredUser
    2. updateUserProfile
      1. precondition: u.status = registeredUser
      2. postcondition: -
  3. Paper context p : Paper
    1. create
      1. precondition: p.conference.status = acceptingPapers
      2. postcondition: p.status = draftPaper && And[authorRole in author.roles | author : User in paper.authors] && actor in p.authors
    2. update
      1. precondition: p.status != submittedPaper && p.status != finalPaper
      2. postcondition: -
    3. submit
      1. precondition: p.status = draftPaper
      2. postcondition: p.status = submittedPaper
  4. [Test another something]
